# 技能管理界面设计

## 你提出的方案分析

✅ **你的想法非常合理！**

让我分析一下两种方案：

### 方案对比

#### 方案 A：纯上传（我之前的设计）
```
用户上传 ZIP → 自动解析 SKILL.md → 提取信息
```
**优点：**
- 快速，只需一步
- 自动化程度高

**缺点：**
- 如果 SKILL.md 格式错误，体验不好
- 用户无法在上传前预览和修改
- 不支持纯文本技能（无脚本）

#### 方案 B：表单 + 上传（你的建议）⭐
```
用户填写表单（名称、描述等）+ 上传 ZIP → 灵活组合
```
**优点：**
- ✅ 用户体验更好，可以预览和调整
- ✅ 支持"纯文本技能"（不上传文件，只填表单）
- ✅ 支持"技能包 + 自定义元数据"（文件为主，表单可覆盖）
- ✅ 错误提示更清晰
- ✅ 可以批量导入（表单导入 JSON/CSV）

**缺点：**
- 需要填写更多字段（但可以做成可选）

#### 2.2 字段说明

| 字段 | 必填 | 说明 | 验证规则 |
|------|------|------|----------|
| **技能 ID** | ✓ | 唯一标识符 | 只能包含字母、数字、连字符 |
| **技能名称** | ✓ | 显示名称 | 2-50 字符 |
| **技能描述** | ✓ | 功能说明 | 10-500 字符 |
| **技能类型** | ✓ | 分类 | tool / workflow / knowledge |
| **版本号** | - | 语义化版本 | 如 1.0.0 |
| **风险等级** | ✓ | 安全等级 | low / medium / high |
| **技能包** | - | ZIP 文件 | 最大 50MB |
| **Markdown 内容** | -* | 技能说明 | 无技能包时必填 |
| **依赖** | - | 其他技能 ID | 逗号分隔 |
| **配置参数** | - | JSON 格式 | 必须是有效 JSON |

**说明：** 如果上传了技能包，Markdown 内容可以从 SKILL.md 自动提取（但允许用户修改）

### 3. 交互逻辑

#### 场景 A：创建纯文本技能（无文件）

```
1. 用户点击"新增技能"
2. 填写表单：
   - 技能 ID: prompt-template
   - 技能名称: 提示词模板
   - 技能描述: 用于...（填写）
   - 技能类型: 知识库
   - 风险等级: 低
   - 技能包: [不上传]
   - Markdown 内容: 填写完整内容
3. 点击"确认创建"
4. 后端：
   - 验证表单
   - 保存到数据库（content 字段）
   - packagePath 为 null
5. 返回成功
```

#### 场景 B：上传技能包（自动提取）

```
1. 用户点击"新增技能"
2. 点击"技能包上传"，选择文件 github.skill
3. 前端自动解析（或上传到后端解析）：
   POST /api/admin/skills/parse-upload
   参数: file (MultipartFile)
   返回: { skillId, name, description, content }
4. 自动填充表单：
   - 技能 ID: github（自动填充，可修改）
   - 技能名称: GitHub 集成（自动填充）
   - 技能描述: 用于...（自动填充）
   - Markdown 内容: SKILL.md 的内容（自动填充）
5. 用户可以修改任何字段
6. 点击"确认创建"
7. 后端：
   - 验证表单
   - 解压技能包到 ~/.luoshen/skills/user/github/
   - 保存到数据库
   - packagePath = ~/.luoshen/skills/user/github/
8. 返回成功
```

```
1. 用户上传技能包 github-partial.skill
```

### 4. 表单验证

#### 前端验证

```javascript
// 技能 ID 验证
const validateSkillId = (id) => {
  const regex = /^[a-z0-9-]+$/;
  if (!regex.test(id)) {
    return '只能包含小写字母、数字和连字符';
  }
  if (id.length < 2 || id.length > 50) {
    return '长度在 2-50 字符之间';
  }
  return null;
};

// 检查技能包或内容二选一
const validateContent = (packageFile, markdown) => {
  if (!packageFile && !markdown) {
    return '请上传技能包或填写 Markdown 内容';
  }
  if (packageFile && markdown) {
    return '技能包和内容不能同时填写';
  }
  return null;
};

// 风险等级联动
const handleRiskLevelChange = (level) => {
  if (level === 'high') {
    showWarning('高风险技能需要管理员审批才能使用');
  }
};
```

#### 后端验证

```java
@PostMapping("/create")
public ResponseEntity<?> createSkill(
    @RequestParam(required = false) String skillId,
    @RequestParam(required = false) String name,
    @RequestParam(required = false) String description,
    @RequestParam(required = false) String type,
    @RequestParam(required = false) MultipartFile file,
    @RequestParam(required = false) String content
) {
    // 1. 基础验证
    if (file == null && StringUtils.isEmpty(content)) {
        return ResponseEntity.badRequest()
            .body("必须上传技能包或填写内容");
    }
    
    // 2. 如果有文件，解压并验证
    if (file != null) {
        ValidationResult result = skillValidator.validate(file);
        if (!result.isValid()) {
            return ResponseEntity.badRequest()
                .body(result.getErrors());
        }
        
        // 自动提取元数据（如果用户没填）
        if (StringUtils.isEmpty(name)) {
            name = result.getMetadata().getName();
        }
    }
    
    // 3. 保存技能
    SkillConfigEntity skill = skillService.create(
        skillId, name, description, type, file, content
    );
    
    return ResponseEntity.ok(skill);
}
```

## ✅ 总结

你的建议非常合理！推荐实现：

1. **表单 + 上传混合模式**
   - 支持纯文本技能（只填表单）
   - 支持技能包上传（自动提取）
   - 支持自定义元数据（手动调整）

2. **分步式表单**
   - 基本信息
   - 技能内容（上传 或 手动填写）
   - 高级选项（可选）

3. **智能辅助**
   - 上传后自动解析
   - 自动填充元数据
   - 实时验证和提示

4. **灵活的 API**
   - 解析上传（预览，不保存）
   - 创建技能（保存）
   - 更新技能（支持版本升级）

这样既保证了灵活性，又提供了良好的用户体验！🎉
