## 二、核心议题

### 2.1 Agent SDK 概述

**核心概念：**
- Agent SDK 是用于构建 AI Agent 的开发框架
- 主要有两种模式：
  1. **流程编排模式** - 通过流程图方式编排 Agent 任务
  2. **自主 Agent 模式** - Agent 自主决策和执行

**平台对比：**

| 平台 | 特点 | 适用场景 |
|------|------|----------|
| Dify | 低代码平台，支持 Agent 编排 | 快速原型开发 |
| 扣子(Coze) | 字节跳动出品，支持多 Agent | 企业级应用 |
| LangChain | 开源框架，灵活度高 | 定制化开发 |
| AutoGen | 微软出品，多 Agent 协作 | 复杂任务场景 |

### 2.2 多 Agent 协作模式

**协作架构：**
```
主 Agent (Master Agent)
    ├── 设计 Agent - 负责 UI/UX 设计
    ├── 开发 Agent - 负责代码实现
    ├── 测试 Agent - 负责质量验证
    └── 部署 Agent - 负责环境部署
```

**关键点：**
- 每个 Agent 有独立的职责和技能
- 主 Agent 负责任务分配和协调
- Agent 之间通过消息传递进行通信

### 2.3 记忆管理 (Memory)

**核心问题：**
- 如何管理长对话历史？
- 如何实现上下文记忆？
- 如何高效检索相关信息？

**解决方案：**
- 使用 Memory MD 存储对话历史
- 实现记忆的增量更新和检索
- 支持短期记忆和长期记忆

### 2.4 MCP 协议

**MCP (Model Context Protocol)：**
- 用于 Agent 与外部工具的交互
- 标准化的工具调用接口
- 支持流式输出

**关键特性：**
- 工具注册和发现
- 参数验证和类型检查
- 结果格式化
