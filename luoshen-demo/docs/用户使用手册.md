## 1. 系统简介

### 1.1 什么是洛神系统？

洛神系统是一个**AI 智能助手管理平台**，让您能够：

- 🤖 **创建和管理 AI 助手** - 根据业务需求定制专属 AI 助手
- 🛠️ **配置 AI 技能** - 为 AI 助手添加各种专业能力
- 💾 **管理对话记忆** - 让 AI 记住重要信息，提供个性化服务
- 📊 **监控使用情况** - 查看系统运行状态和统计数据

### 1.2 系统访问

**访问地址：** 由管理员提供（例如：http://192.168.1.100:5174）

**推荐浏览器：**
- Chrome（推荐）
- Edge
- Firefox
- Safari

> ⚠️ **提示：** 首次访问可能需要输入用户名和密码，请联系管理员获取

## 3. 功能介绍

### 3.1 仪表板（Dashboard）

**功能说明：** 查看系统整体运行状态

**显示内容：**
- **Agent 统计** - 已创建的 AI 助手数量
- **Skill 统计** - 已配置的技能数量
- **会话统计** - 当前活跃的对话数量
- **记忆统计** - 系统存储的记忆条数
- **系统状态** - 运行状态指示

**使用场景：**
- 快速了解系统使用情况
- 监控系统健康状态

---

### 3.2 Agent 管理

**功能说明：** 创建和管理 AI 助手

#### Agent 类型说明

| 类型 | 图标 | 说明 | 适用场景 |
|------|------|------|----------|
| **LEADER** | 👑 | 主控 Agent | 负责任务分发和协调 |
| **CORE** | ⭐ | 核心 Agent | 处理核心业务逻辑 |
| **SUB** | 🔧 | 子 Agent | 执行具体专项任务 |

#### Agent 字段说明

| 字段 | 说明 | 是否必填 | 示例 |
|------|------|----------|------|
| Agent ID | 唯一标识符 | ✅ 是 | customer-service-001 |
| 名称 | 显示名称 | ✅ 是 | 客服助手 |
| 描述 | 功能描述 | ❌ 否 | 处理客户咨询问题 |
| 系统提示词 | AI 的行为指令 | ❌ 否 | 你是一名专业的客服... |
| 类型 | Agent 类型 | ✅ 是 | LEADER |
| 模型名称 | 使用的 AI 模型 | ❌ 否 | qwen-max |
| 工具 | 可调用的工具 | ❌ 否 | 搜索、天气查询 |
| 技能 | 拥有的技能 | ❌ 否 | 产品知识、FAQ |
| 是否启用 | 启用状态 | ✅ 是 | 是/否 |

---

### 3.3 Skill 管理

**功能说明：** 管理AI 助手的技能

#### Skill 类型说明

| 类型 | 说明 | 示例 |
|------|------|------|
| **TOOL** | 工具型技能 | 天气查询、翻译工具 |
| **WORKFLOW** | 工作流技能 | 订单处理流程、审批流程 |
| **KNOWLEDGE** | 知识型技能 | 产品知识库、FAQ 知识 |

#### Skill 字段说明

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| Skill ID | 唯一标识符 | ✅ 是 |
| 名称 | 技能名称 | ✅ 是 |
| 描述 | 技能描述 | ❌ 否 |
| 内容 | Markdown 格式的技能内容 | ❌ 否 |
| 类型 | 技能类型 | ✅ 是 |
| 所属 Agent | 关联的 Agent | ❌ 否 |
| 版本 | 版本号 | ❌ 否 |
| 是否启用 | 启用状态 | ✅ 是 |

---

### 3.4 Session 管理

**功能说明：** 查看和管理对话会话

#### Session 状态说明

| 状态 | 说明 |
|------|------|
| **active** | 活跃状态，正在进行对话 |
| **paused** | 暂停状态，暂时中断 |
| **closed** | 已关闭，对话结束 |

#### Session 信息包含

- 用户 ID
- 使用的 Agent
- 消息数量
- 创建时间
- 最后活跃时间

---

### 3.5 Memory 管理

**功能说明：** 管理 AI 的记忆存储

#### Memory 类型说明

| 类型 | 说明 | 保留时间 |
|------|------|----------|
| **short_term** | 短期记忆 | 当前会话有效 |
| **long_term** | 长期记忆 | 长期保存 |
| **semantic** | 语义记忆 | 永久保存 |

#### Memory 重要性评分

- **0.0 - 0.3**：低重要性，可能被清理
- **0.4 - 0.7**：中等重要性，正常保留
- **0.8 - 1.0**：高重要性，优先保留

---

### 3.6 MCP 工具管理

**功能说明：** 管理可调用的外部工具

#### 工具类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **query** | 查询类工具 | 数据库查询、搜索 |
| **action** | 动作类工具 | 发送邮件、创建订单 |
| **control** | 控制类工具 | 系统控制、配置修改 |

## 4. 操作指南

### 4.1 创建 AI 助手

**步骤：**

1. 点击左侧菜单 **"🤖 Agent"**
2. 点击右上角 **"创建 Agent"** 按钮
3. 填写 Agent 信息：

```
Agent ID: my-assistant-001
名称: 我的助手
描述: 日常工作助手
类型: LEADER
系统提示词: 你是一个友好的工作助手，帮助用户处理日常任务
是否启用: 是
```

4. 点击 **"保存"** 按钮

**效果：** 创建成功后，可在列表中看到新的 Agent

---

### 4.2 为 Agent 添加技能

**步骤：**

1. 点击左侧菜单 **"🛠️ Skill"**
2. 点击右上角 **"创建 Skill"** 按钮
3. 填写技能信息：

```
Skill ID: skill-greeting
名称: 问候技能
描述: 处理用户的问候和寒暄
类型: WORKFLOW
内容: 
  # 问候技能
  
  ## 触发条件
  - 用户说"你好"、"在吗"等问候语
  
  ## 响应方式
  - 友好地回应用户
  - 询问有什么可以帮助的
  
所属 Agent: my-assistant-001
是否启用: 是
```

4. 点击 **"保存"** 按钮

---

### 4.3 查看对话记录

**步骤：**

1. 点击左侧菜单 **"💬 会话"**
2. 浏览会话列表
3. 点击某条会话查看详情

**可执行操作：**
- 查看会话状态
- 查看消息数量
- 更新会话状态（暂停/关闭）
- 删除会话

---

### 4.4 搜索记忆内容

**步骤：**

1. 点击左侧菜单 **"🧠 记忆"**
2. 在搜索框输入关键词
3. 点击 **"搜索"** 按钮
4. 查看搜索结果

---

### 4.5 编辑和删除

**编辑操作：**
1. 在列表中找到目标项目
2. 点击 **"编辑"** 按钮（或双击行）
3. 修改信息
4. 点击 **"保存"**

**删除操作：**
1. 在列表中找到目标项目
2. 点击 **"删除"** 按钮
3. 确认删除操作

> ⚠️ **警告：** 删除操作不可恢复，请谨慎操作！

## 5. 常见场景

### 场景一：创建客服助手

**目标：** 创建一个处理客户咨询的 AI 助手

1. **创建 Agent**

   - 系统提示词: `你是一名专业的客服人员，友好、耐心地解答客户问题`

   - 添加产品搜索工具

**目标：** 创建一个帮助编程的 AI 助手

## 6. 常见问题

### Q1: 忘记密码怎么办？

**A:** 请联系系统管理员重置密码。

---

### Q2: Agent 创建后不生效？

**A:** 请检查：
1. Agent 是否已"启用"
2. 必填字段是否完整
3. 尝试刷新页面

---

### Q3: 如何让 AI 更智能？

**A:** 可以通过以下方式提升 AI 能力：
1. 编写详细的系统提示词
2. 添加更多专业技能
3. 配置实用的工具
4. 积累有价值的记忆

---

### Q4: 记忆太多会影响性能吗？

**A:** 系统会自动管理记忆：
- 短期记忆会自动清理
- 长期记忆根据重要性保留
- 可以手动删除不需要的记忆

---

### Q5: 如何批量导入技能？

**A:** 目前需要逐个创建。如需批量导入，请联系技术支持。

---

### Q6: 系统响应很慢怎么办？

**A:** 可能的原因：
1. 网络问题 - 检查网络连接
2. 服务器负载高 - 稍后再试
3. 数据量过大 - 清理不必要的数据

## 7. 技巧与最佳实践

### 7.1 编写好的系统提示词

**模板：**
```
你是[角色描述]。

你的主要职责是：
1. [职责1]
2. [职责2]
3. [职责3]

你的沟通风格：
- [风格特点1]
- [风格特点2]

注意事项：
- [限制1]
- [限制2]
```

**示例：**
```
你是一名专业的金融顾问。

你的主要职责是：
1. 解答客户关于理财的问题
2. 提供投资建议
3. 解释金融概念

你的沟通风格：
- 专业但易懂
- 客观中立
- 耐心细致

注意事项：
- 不做具体的投资承诺
- 提醒用户投资有风险
- 建议用户咨询专业人士
```

---

### 7.2 技能内容编写规范

**推荐格式（Markdown）：**

```markdown
# 技能名称

## 功能描述
简要描述这个技能的作用

## 触发条件
- 条件1
- 条件2

## 执行步骤
1. 步骤1
2. 步骤2
3. 步骤3

## 输出格式
描述预期的输出格式

## 注意事项
- 注意点1
- 注意点2
```

---

### 7.3 记忆管理建议

**应该记住的信息：**
- ✅ 用户偏好和习惯
- ✅ 重要的业务规则
- ✅ 常见的问答对
- ✅ 用户的基本信息

**不需要记住的信息：**
- ❌ 临时性的对话
- ❌ 无关的闲聊
- ❌ 错误的或过时的信息

---

### 7.4 定期维护

**建议每周检查：**
1. 清理无用的 Agent
2. 更新过时的技能
3. 删除不必要的记忆
4. 检查系统状态
